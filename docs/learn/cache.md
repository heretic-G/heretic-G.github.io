#### Freshness 新鲜期
    也就是一个资源能保存多久 只有Expires和 Cache-Control: max-age 直接声明这个时间长度
    也就是如果是新鲜的就直接使用缓存 而不是在去请求也就是 常说的强制缓存(所以从逻辑看这里只是命中缓存 感觉并没有强制的意思)

#### Validation
    Last-Modified / If-Modified-Since
    Etag / If-None-Match

#### Cache-Control
    1.1增加内容
    max-age     缓存有效期 如果存在Age还需要减掉Age
    s-maxage    只对代理服务器有效 覆盖max-age和Expires 
    max-stale   即使缓存失效但是失效时间小于这个时间客户端也可以接受 这个主要是中间的返回在请求中增加
    min-fresh   超过min-fresh的都不接受

    public      一般默认就是他谁都可以缓存
    private     中间不能缓存只有client可以缓存

    no-cache    代表缓存没有新鲜度 每次都需要去校验
    no-store    代表了不会发生任何缓存的事件 每次都去获取新的

    must-revalidate 每次都需要校验缓存有效性
    no-transform 不能变更缓存 有一些代理服务器会压缩或者其他的处理 这里会禁止
    immutable   不可变 在支持的浏览器中 很多操作会导致max-age还没到时间触发验证  不可变告诉浏览器 这个资源不会发生变化不需要校验
    

#### Expires
    1.0增加的内容
    缓存的有效期时间
    但是因为客户端和服务器的时钟不一致 可能导致client有限期和实际期望有差异

#### Last-Modified
    1.0增加内容
    这个字段只有他生效的时候 会有个启发式缓存这里会利用他和Date计算差值*10% 
    在这个时间内客户端直接使用缓存 超过会去服务器验证一下
    Cache-Control 当不使用age的时候 可以控制验证的实际也就是他可以延长 也可以每次触发去验证

    所以他第一生成时间是秒的 也就是秒以内的颗粒度胺无法感知 第二就是一个长时间不变更资源 在变更后可能有一段时间不能变更
    这个和浏览器的启发式新鲜度算法相关

#### Etag
    1.1增加内容
    根据服务器生成的值去再次请求看服务器资源有没有变更 这个值的麻烦在于 服务器需要计算文件的值
    (当然可以类似MD5 也可以直接搞个变更时间 文件名啥的 这个随便定)

#### Date
    响应报文创建时间 主要是和Last-Modified一块使用

#### Age
    好像是在代理服务器的时长 也就是一个缓存如果设置max-age这里需要减少代理服务器的时长以能够比较精确的去判断是不是到了失效时间
    不然多层可能导致max-age的时效变为几倍 导致无法穿透缓存

#### Pragma
    1.0 中才能使用 只有no-cache 也就是每次都回去服务端校验

#### Vary 
    主要是约束代理服务器做缓存的时候根据vary的list依次去缓存 以区别不同的客户端的需求
    举个例子 相同资源请求地址 A在request中说我只接受A1编码的数据
    B在request中我只接受B1编码的数据 这时候 代理服务器如果不区分 都返回一样的 会导致另一方不能使用 发生错误
    这时候Vary就告诉代理服务器 你要基于编码字段来区分下缓存 那这时候类似缓存的key就不是url还有就是编码 一起当key
    如果没有命中就会去服务器取 命中返回

    Vary:* 就是任何头信息都可能是判断依据 而且可能包括不含有头的信息 不能缓存任何(这是看到别人翻译的 我具体没看)

#### 启发式缓存
    就是Last-Modified和Date 根据已经缓存的时候来确定新鲜度的长度
